CSS Mixins
  • A mixin is a reusable block of CSS declarations you can include in multiple places.
  • Native CSS doesn’t support mixins, but pre-processors like SASS/SCSS do.
@mixin center {
  display: flex;
  justify-content: center;
  align-items: center;
}
.box {
  width: 100px;
  height: 100px;
  @include center;
}
Pseudo-classes
  • A pseudo-class defines the special state of an element (hovered, focused, visited, etc.).
  • :hover, :focus, :active, :nth-child(n), :not(selector), :disabled, :checked, :required
a:hover {
  color: red;
}
input:focus {
  border-color: blue;
}
li:first-child {
  font-weight: bold;
}

postions in css
  • the position property defines how an element is placed in the document flow and how it can be moved using top, right, bottom, and left.
static (default)
  • Normal document flow (no positioning).
  • top, left, etc. have no effect.
relative
  • Positioned relative to its normal position.
  • Space is still reserved for it in the document.
  • You can shift it using top, left, etc.
absolute
  • Positioned relative to the nearest positioned ancestor (non-static).
  • If no such ancestor, it’s positioned relative to the html (viewport).
  • Removed from document flow (other elements ignore its space).
fixed
  • Positioned relative to the viewport (browser window).
  • Doesn’t move on scroll.
  • Useful for sticky headers, buttons, etc.
sticky
  • Hybrid of relative and fixed.
  • Behaves like relative until it hits a scroll threshold, then becomes fixed.
  • Useful for sticky headers, buttons, etc.
