What is Jenkins

Jenkins is an open-source automation server used to build, test, and deploy software. Think of it as your automation butler who runs the steps you normally do manually ‚Äî like pulling code, installing dependencies, running tests, building apps, and deploying them ‚Äî automatically whenever code changes.

What is Docker?

Docker is a platform that lets you package your app and everything it needs (Node, Angular CLI, dependencies, etc.) into one lightweight, portable container.

Basic Docker Use Case for Angular
ng build --configuration=production
//Create a Dockerfile
# Use Nginx to serve the Angular build
FROM nginx:alpine
COPY ./dist/my-angular-app /usr/share/nginx/html
//Build the Docker Image
docker build -t my-angular-app .
//Run It!
docker run -p 8080:80 my-angular-app
//access it on http://localhost:8080
CI/CD Configuration
Where Is It Configured?
  • CI/CD is typically configured using a YAML file in your repository
  • GitHub Actions: .github/workflows/deploy.yml
  • Azure Pipelines: azure-pipelines.yml
  • Jenkins: Uses Jenkinsfile or pipeline GUI
  • Bitbucket Pipelines: bitbucket-pipelines.yml
